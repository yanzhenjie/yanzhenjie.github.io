No rule to make target pyconfig.h', needed byPrograms/python.o’

我日常使用 pyenv 管理不同的 Python 版本，最近在 MacOS 13.2 测试 IndexTTS 2 时发现编译环境需要升级（由于本文遇到的问题我怀疑不用升级） ，最近由 MacOS 13.2 升级到 MacOS 15.7 后，需要重装 pyenv 中的几个 python 版本。

我的环境：

  • MacOS Sequia 15.7
  • Command_Line_Tools_for_Xcode_16.4_MacOS_15.3
  • Homebrew 4.6.15
  • pyenv 2.6.8

我的问题表现：

这里的错误是：No rule to make target pyconfig.h', needed byPrograms/python.o’。

然后在 pyenv 项目下和 StackOverflow 找了很多资料做了一些尝试，我做过哪些尝试？

  • Wiki，这里的很多细节还是值得一看：

    1. 参考https://github.com/pyenv/pyenv/wiki#suggested-build-environment，每一项都做了检查和尝试，均遇到同样的问题。
    2. 参考https://github.com/pyenv/pyenv/wiki/Common-build-problems，每一项都做了检查和尝试，均遇到同样的问题。

  • StackOverflow：

    1. https://stackoverflow.com/questions/51551557/pyenv-build-failed-installing-python-on-macos

  • Issues：

    1. https://github.com/pyenv/pyenv/issues/1354，问题相同，但是我没有理解答案，忽略答案提到的一个命令，没有做检查。
    2. https://github.com/pyenv/pyenv/issues/3310，问题类似，但没有解决。
    3. https://github.com/pyenv/pyenv/issues/3062，在这里找到了答案，但历程有点揪心。

最终是 Issues 中的 1 和 3 解决了我的问题，我接下来说下原因。

最开始，我是在 Issues 中找答案的，所以一直在看 Issues，但是其实问题#1354中已经讲到了，但是没有引起我的重视：

Problem solved. I had overridden the diff command in my .bash_profile. Uncommenting those lines make the ./configure step detect that pyconfig.h needed to be created!

我在~/.zshrc、/etc/zshrc、/etc/zprofile、/etc/paths中都找了有没有什么配置覆盖了 diff 命令，但是没有找到。

接下来我就有点郁闷了，就继续尝试了 Wiki、StackOverflow 中提到的一些方案，浪费了个把小时的时间吧。

就在刚辞，然后回到问题#3062中再过一遍，发现成员提到：

This is where it goes wrong. Need to closely examine what happens at this moment: what is your and , if the file and look normal, what the command outputs, what code the returns.awk diff .awk pyconfig.h.in awk diff
Perhaps we could arrange a remote access session for me to examine that on your system? Contact me on the e-mail in my profile if you’re interested.
